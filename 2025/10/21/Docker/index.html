4. docker网络

默认情况下，所有容器都是以bridge的方式连接到docker的一个虚拟网桥上，分配给容器的ip 前16位是不变的

使用ip addr可以查看本机网络配置，我们可以查看到docker0，也就是默认虚拟网桥

而常用的docker网络是使用自定义网桥，加入自定义网桥的容器之间可以通过容器名相互访问

此时，我们就可以将容器加入到新创建的网段当中，以mysql容器为例

加入之后我们查看mysql 容器的详细信息就可以发现， 除了bridge网段之外，mysql容器加入了新的网段

同样可以在docker run 的时候可以加入--network字段，创建之后就链接到指定网桥（注意这个时候该容器就不会出现在默认网桥上）

此时我们可以进入mysql 容器当中，使用容器名dd访问dd容器

5.部署项目

使用Maven中的package进行打包，tips：显示[ERROR] The specified user settings file does not exist: C:\Users\Lenovo\.m2\settings.xml

代表我们是setting.xml不在C盘下，解决方案有两种

  1. 拷贝**%M2_HOME%/conf/settings.xml的文件到%USER_HOME%/.m2/settings.xml**中即可。
  2. 进入Maven设置更改user setting file的指向为**%M2_HOME%/conf/settings.xml**即可

同时在package build的时候，如果出现中文路径报错乱码可以进行以下更改，在Maven runner中设置VM Options -Dfile.encoding=GB2312

将打包文件的docker file和jar包放入服务器上
